# Code of Conduct

CompText MCP is intended to be a useful and welcoming project for developers working with OpenCode, Rust, MCP, and deterministic context tooling.

## Expected behavior

Participants are expected to:

- be respectful and constructive
- focus on technical evidence
- explain issues with reproducible examples when possible
- accept that maintainers may reject changes that weaken safety or determinism
- help new contributors understand the project boundaries

## Unacceptable behavior

The following is not acceptable:

- harassment or personal attacks
- discriminatory language
- spam or low-effort promotion
- pressuring maintainers to add unsafe features
- requests to bypass safety defaults or secret protections

## Enforcement

Maintainers may edit, hide, or remove comments, issues, or pull requests that violate this code of conduct. Repeated violations may lead to blocking from the project.

## Reporting

Open a private security report for security-sensitive issues. For general conduct problems, contact the repository owner through GitHub.
